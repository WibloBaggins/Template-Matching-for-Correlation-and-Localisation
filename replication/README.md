# Replication Package — Template Matching for Multi-Sensor Localisation

**Thesis:** *Template Matching for Correlation and Localisation of Multiple Sensor Types in Unstructured Environments for Traversability Assessment*
**Author:** William Buchan — MSc by Research, Cranfield University (2024)

This folder contains everything needed to replicate the two core experiments from the thesis. All raw data is included, with scripts to regenerate every intermediate product from scratch.

---

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. (Optional) Regenerate ALL derived data from raw inputs
#    This creates edge-detected images + heightmaps from the raw point cloud CSVs.
#    Pre-computed data is already included, so skip this if you just want results.
python generate_all_data.py

# 3. Run the UE5 simulated localisation test (main thesis result)
#    Produces: statistical plots + match visualisations with red/white overlays
python run_ue5_test.py

# 4. Run the generalised noise-robustness test
python run_generalised_test.py
```

All scripts must be run from **this directory** (they use relative paths to `data/`).

## What Each Script Does

### `generate_all_data.py` — Full Data Generation Pipeline

Regenerates every intermediate product from the two raw inputs:

**Step 1 — Overhead edge image:** Applies Canny edge detection to the raw greyscale overhead image (`greyscale_1_pixel_per_square_meter.png`) to produce the target edge map.

**Step 2 — Radar heightmaps:** Reads the 127 formatted point cloud CSVs from the loop_5 driving path, accumulates them in 5-second windows, and for each window:
1. Rasterises the point cloud to a 1m/pixel heightmap
2. Interpolates gaps using scipy linear griddata
3. Computes steepness (gradient magnitude): `steepness = sqrt(dz/dx² + dz/dy²)`
4. Computes steepness-of-steepness (second derivative)
5. Applies Canny edge detection to all three, saving masked and unmasked variants
6. Writes corner_info metadata (bounding box coordinates + unmasked area count)

Produces 25 timesteps × 10 subdirectories = 250 images + 25 corner_info files.

### `run_ue5_test.py` — Simulated Radar Localisation (Chapter 5)

The primary thesis result. Answers: *can a vehicle localise itself on a satellite image using only edge-detected radar heightmaps and template matching?*

**Outputs** (saved to `output/`):

1. **`results_error_vs_area.png`** — Statistical scatter plot: localisation error vs known scanned area, with linear regression trendlines per pre-processing method. Secondary axis shows percentage of total target area.

2. **`match_visualisations/`** — For the best match of each method:
   - **Left panel:** Full 2000x2000 overhead map with the matched region highlighted as a green rectangle. Red cross = matched origin, cyan cross = true origin, orange line = error vector.
   - **Right panel:** Zoomed-in overlay of the matched region. The overhead edge map appears in white/black, with the radar edge map pixels coloured **red** on top. This directly shows how the radar "fingerprint" aligns with the satellite features.

### `run_generalised_test.py` — Noise Robustness (Chapter 4)

Tests noise tolerance by progressively masking template pixels. Four masking methods (random, boundary, central, block) at 0-100% in 1% increments, 10 trials each. Runtime ~15-30 minutes.

## Core Algorithm

All scripts use the same fundamental approach:

1. **Normalise** all sensor data to 1 meter per pixel 2D images
2. Apply **Canny edge detection** to extract structural features
3. Run OpenCV's **`cv2.matchTemplate(target, template, cv2.TM_CCOEFF_NORMED)`** — normalised cross-correlation
4. The location maximising the correlation coefficient is the best match
5. Convert pixel coordinates to real-world meters using known coordinate metadata
6. Compute **Euclidean distance error** between matched and true positions

No deep learning, no labelled training data, no sensor-specific architectures.

---

## Key Equations

**Normalised Cross-Correlation (TM_CCOEFF_NORMED):**

```
R(x,y) = Σ[T'(x',y') · I'(x+x', y+y')] / sqrt(Σ[T'²] · Σ[I'²])
```

Where T' and I' are zero-mean template and image patches.

**Steepness (gradient magnitude):**

```
steepness(x,y) = sqrt((dz/dx)² + (dz/dy)²)
```

**Heightmap rasterisation:**

```
grid_cell(x,y) = z  where  (x,y) = round(point_x - min_x, point_y - min_y)
```

**Interpolation:** scipy.interpolate.griddata with method='linear' (Delaunay triangulation).

**Distance error:**

```
error = sqrt((matched_x - true_x)² + (matched_y - true_y)²)
```
